# tcgapi-mcp

`tcgapi-mcp` is a local MCP server for trading card game data from the TCG Tracking API. It is designed to run over stdio so tools like Claude Desktop can call it as a local server.

This server currently exposes:

- 5 tools for categories, set search, products, pricing, and SKU data
- 2 concrete resources
- 4 resource templates
- 3 prompts

## What It Does

Use `tcgapi-mcp` when you want an MCP client to answer questions like:

- "What sets exist for Pokemon?"
- "Find the set for Surging Sparks."
- "Show the sealed products for this set."
- "Get pricing for this set or product."
- "Show SKU-level pricing by condition, language, and variant."

The server talks to the upstream TCG Tracking API and returns structured JSON that works well for MCP hosts and LLM clients.

## Installation

Release builds are published as platform-specific archives generated by Goreleaser:

- `tcgapi-mcp_<version>_darwin_amd64.tar.gz`
- `tcgapi-mcp_<version>_darwin_arm64.tar.gz`
- `tcgapi-mcp_<version>_linux_amd64.tar.gz`
- `tcgapi-mcp_<version>_linux_arm64.tar.gz`
- `tcgapi-mcp_<version>_windows_amd64.zip`
- `tcgapi-mcp_<version>_windows_arm64.zip`

Install the archive for your platform, extract it, and place the `tcgapi-mcp` binary somewhere on your `PATH`.

Example for macOS or Linux:

```bash
tar -xzf tcgapi-mcp_<version>_<os>_<arch>.tar.gz
chmod +x tcgapi-mcp
mv tcgapi-mcp ~/.local/bin/tcgapi-mcp
```

Example for Windows:

- download the matching `.zip` archive
- extract `tcgapi-mcp.exe`
- place it in a directory on your `PATH`

Verify the binary:

```bash
tcgapi-mcp --version
```

If you want to build from source instead of downloading a release:

```bash
make build
```

That produces `./bin/tcgapi-mcp`.

## Use It In An MCP Host

`tcgapi-mcp` is a stdio MCP server. Any MCP host that supports launching a local stdio server can use it by pointing at the installed binary.

Generic stdio server settings:

- command: `tcgapi-mcp`
- args: none
- env: optional `TCG_CACHE_DIR`, `TCG_LOG_LEVEL`, `TCG_API_TIMEOUT`, `TCG_PAGE_SIZE`

For hosts that want an absolute binary path instead of a command name, point them to the installed executable directly.

### Claude Desktop

Add the server to Claude Desktop in your `claude_desktop_config.json` file.

Example configuration:

```json
{
  "mcpServers": {
    "tcgapi-mcp": {
      "command": "tcgapi-mcp",
      "args": [],
      "env": {}
    }
  }
}
```

With cache persistence:

```json
{
  "mcpServers": {
    "tcgapi-mcp": {
      "command": "tcgapi-mcp",
      "args": [],
      "env": {
        "TCG_CACHE_DIR": "/absolute/path/to/cache-dir"
      }
    }
  }
}
```

If Claude Desktop does not resolve binaries from your `PATH`, use an absolute path for `command` instead.

After saving the config, restart Claude Desktop and ask it to use the server. Good smoke-test prompts:

- "List the available TCG categories."
- "Search Pokemon sets for Surging Sparks."
- "Get products for set ID 41036 in Pokemon."
- "Get pricing for that set."

### Other MCP Hosts

Use the same stdio settings in any other MCP client:

- command: `tcgapi-mcp`
- args: `[]`
- env: optional runtime variables such as `TCG_CACHE_DIR`

## Configuration

The server is configured with environment variables:

| Variable | Default | Purpose |
| --- | --- | --- |
| `TCG_API_URL` | `https://tcgtracking.com/tcgapi/v1` | Upstream API base URL |
| `TCG_API_TIMEOUT` | `30s` | HTTP timeout for upstream requests |
| `TCG_CACHE_DIR` | unset | Enables cache persistence across restarts |
| `TCG_CACHE_MAX_MB` | `256` | In-memory cache size limit |
| `TCG_LOG_LEVEL` | `info` | Log verbosity: `debug`, `info`, `warn`, `error` |
| `TCG_PAGE_SIZE` | `50` | Default page size for paginated product results |

Important runtime behavior:

- MCP protocol traffic uses stdout
- logs go to stderr
- `TCG_CACHE_DIR` is optional but useful for repeated local use

## MCP Surface

Tools:

- `list_categories`
- `search_sets`
- `get_set_products`
- `get_set_pricing`
- `get_set_skus`

Resources:

- `tcg:///meta`
- `tcg:///categories`

Resource templates:

- `tcg:///{categoryId}/sets`
- `tcg:///{categoryId}/sets/{setId}`
- `tcg:///{categoryId}/sets/{setId}/pricing`
- `tcg:///{categoryId}/sets/{setId}/skus`

Category arguments for tools accept:

- numeric category IDs
- category names like `Pokemon`
- aliases like `mtg`

## Local Development

Build:

```bash
make build
```

Test:

```bash
make test
```

Validate the checked-in OpenAPI contract:

```bash
make openapi-validate
make generate-check
```

Run the server directly over stdio:

```bash
make run
```

Make targets default to `GOCACHE=/tmp/tcgapi-mcp-go-cache` in this environment.

## Notes

- The server currently supports stdio transport only.
- Generated OpenAPI bindings are derived from `openapi/tcgtracking-openapi.yaml`.
- Release automation exists, but the main user path today is to run the local binary as an MCP server.
